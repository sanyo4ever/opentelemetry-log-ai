# OpenTelemetry Log AI

## Опис проекту
Проект спрямований на обробку логів і телеметрії з понад 1000 хостів (як Windows, так і Linux) за допомогою сервісу OpenTelemetry. Логи збираються через сервер логів Signoz, який використовує OpenTelemetry Collector і відправляє дані в базу даних ClickHouse.

### Бізнес задача
Основна мета проекту - створити сервіс, який в онлайн-режимі:
- Сканує логи, що зберігаються в ClickHouse.
- Розуміє суть цих логів та шукає сигнатури, які можуть становити небезпеку.
- У разі виявлення небезпечних логів, надсилає алерт у сервіс Keep через webhook.

Також планується використання готових правил та сигнатур із проекту [Sigma Rules](https://github.com/SigmaHQ/sigma).

## Архітектура проекту
Проект використовує модульну архітектуру:

1. **Збір логів:**
   - Використовується OpenTelemetry для збору телеметрії та логів із серверів Windows та Linux.
   - Встановлений OpenTelemetry Collector пересилає дані до сервера обробки Signoz.
   
2. **Збереження даних:**
   - Логи та телеметрія агрегуються та зберігаються в базі даних ClickHouse для подальшої обробки.

3. **Аналіз логів:**
   - Логи з бази даних обробляються за допомогою процесора, який:
     - Мапить правила із Sigma Rules на стандарт OpenTelemetry.
     - Аналізує їх та обробляє на основі визначеної логіки.
   - У разі необхідності обробка може бути переведена у стандарт SIEM для інтеграції з інтерфейсами Sigma Rules.

4. **Система алертів:**
   - У разі декодування небезпечних логів або сигнатур, сервіс:
     - Формує сповіщення.
     - Надсилає алерт до сервісу Keep через webhook-інтеграцію.

## Потенційні подальші кроки
- Дослідження готових відкритих інтерфейсів використання Sigma Rules разом із SIEM стандартами.
- Зміна структури даних у ClickHouse для інтеграції із SIEM стандартами та автоматичне використання правил Sigma.
- Оптимізація продуктивності сервісу для обробки великих обʼємів логів в реальному часі.
